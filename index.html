Matériel

Jour par jour

Jour 1

Brainstorming : Produire un maximum d’idées variées pour explorer les différentes pistes.

Jour 2

Maquettage: Après le brainstorming, le maquettage permet de visualiser et de concrétiser les idées sélectionnées. Nous avons commencé la création du site web et d'un compte GitHub qui va nous permettre de travailler à deux sur le site. Nous avons eu des difficultées pour comprendre github et son langage git, à chaque modification on devait créer une branch. Et pendant ce temps le deuxième groupe est sur la concéption, ils galèrent avec arduino et les branchements.

Jour 3

Nous éprouvons grande difficulté à comprendre github nous avons des conflits entre nos deux codes. La cause: plusieurs lignes de code divergentes. Petit problème de connecteurs wago du côté de l'équipe de conception.

Jour 4

C'est le stress: l'équipe conception n'arrive toujours pas à comprendre pourquoi ça ne marche pas, ça dure toute la matinée, finalement c'était un problème dans le code, nous avions modifié une partie du code pour tenter de le faire fonctionner, c'est ce qui finalement l'avait cassé. L'après-midi on commence la mise en place du robot: on fait la découpeuse et on tente d'assembler les pieces les unes avec les autres, c'est compliqué on a pas assez réflechi à la question avant cette étape et mettre le arduino et sa batterie dans le robot s'avère complexe. La manette reste problématique on avait pas anticipé les trous de la découpeuse avec nos potentionmètres.
